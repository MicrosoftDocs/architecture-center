## How to implement the Reliable Web App pattern

Find the specific guidance that you need in the following sections of this article:

- [Business context](#business-context): Align this guidance with your business context and learn to define immediate and long-term goals that drive replatforming decisions.

- [Architecture guidance](#architecture-guidance): Select the right cloud services and design an architecture that meets your business requirements.

- [Code guidance](#code-guidance): Implement the Retry, Circuit Breaker, and Cache-Aside design patterns to improve the reliability and performance efficiency of your web app in the cloud.

- [Configuration guidance](#configuration-guidance): Configure authentication and authorization, managed identities, rightsized environments, infrastructure as code (IaC), and monitoring.
